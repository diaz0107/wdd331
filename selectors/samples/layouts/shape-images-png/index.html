Using Images as Shapes: A PNG with Alpha Transparency

This demo only works when the browser can create an origin. You may need to run a local server for this demo.

A Good Cuppa Joe

It's also possible to use an image as a shape with shape-outside. You can use any image with an alpha channel. This includes PNG images, SVG images with HSLA or RGBA fill values, and CSS gradients.

It does not, however, include image formats such as GIF, that only support binary transparency. Such formats are incompatible with shape-outside.

Although we've added a shape-outside declaration to an img element, the shape of the image is separate from its visibility.

Without the shape-outside declaration, content would flow around the edges of the img element's box. With the shape-outside declaration, however, adjacent content flows around the transparent areas of the PNG image.

Use the check box above to disable or enable the shape-outside declaration. Notice how the text wrapping adjusts.

When using image URLs with shape-outside, the browser makes a potentially CORS-enabled fetch for the resource. The browser must be able to resolve the document path in such a way that it can generate an origin. The document origin is included among the request headers that accompany the browser's request for the image.
