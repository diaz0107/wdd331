Help With Demos

Several of these demos follow the Same-Origin Policy.

An origin is a string value based on the document's URL. It consists of three parts:

  • the protocol, typically http:// or https://;
  • the host name, which often matches the domain or sub-domain portion of the URL; and
  • the port number.

Under a Same-Origin Policy, images, scripts, fonts, and media files need to:

  • share the same origin as the requesting document; or
  • be served with an Access-Control-Allow-Origin response header that allows the requesting origin to load the requested URL.

If the browser can't create an origin for the requesting document, it won't request or load external assets. Most browsers will not create an origin for the requesting when you load a document directly from your file system.

To fix this, use a local webserver. Some operating systems and programming languages include a web server that's perfect for these purposes.

  • If you have PHP installed: php -S localhost:8000

  • If you have Python 3 installed: python -m http.server 8000 --bind 127.0.0.1

MAMP/MAMP Pro and XAMPP are other options.
